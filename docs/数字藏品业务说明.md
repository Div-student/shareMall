# 数字藏品业务说明

> 版本：v1.0（2026-06-13）  
> 与当前代码实现对齐：`apps/server/src/modules/nft`（`NftService`、`NftTradeService`、`NftPriceService`、`NftConfigService`）

---

## 一、业务概览

数字藏品（电子 IP）支持两条链路：

| 链路 | 说明 | 支付方式 |
| --- | --- | --- |
| **一级兑换** | 平台发行库存，用户用可用贡献金兑换 | 扣 `available_fund` |
| **二级 C2C** | 持有人挂单，他人购买 | 买家扣 `available_fund`；卖家所得入 `withdrawable_cash` |

两类交易均基于**动态参考价**定价，而非固定「兑换贡献金」或卖家自由喊价。

---

## 二、价格体系

### 2.1 三类价格字段（`nft` 表）

| 字段 | 说明 |
| --- | --- |
| `start_price` | **起始价格**，后台发行/编辑时设置 |
| `current_price` | **当前参考价**，每日波动更新；展示给用户 |
| `exchange_fund` | 兼容字段，与 `current_price` 同步（历史接口字段） |
| `last_price_date` | 上次日波动更新日期 |

### 2.2 每日价格波动

- **触发时机**：访问藏品相关接口时检查（懒更新）；若 `last_price_date` 非当日则更新。
- **算法**：在 `±dailyFluctuationPct` 内随机波动（默认 ±5%）  
  `newPrice = round(currentPrice × (1 + random(-1~1) × dailyFluctuationPct))`，最低 0.01。
- **历史记录**：写入 `nft_price_history`（`price_date`、`price`、`change_pct`），供详情页走势图。

### 2.3 成交价格（兑换 & 二级市场购买）

服务端在成交瞬间计算，公式：

```
dealPrice = round(currentPrice × (1 + random(0,1) × dealPremiumPct))
```

| 项 | 说明 |
| --- | --- |
| `currentPrice` | 成交时藏品当前参考价 |
| `dealPremiumPct` | 后台配置成交溢价比例（默认 0.1 = 最高溢价 10%） |
| `random(0,1)` | 服务端随机数，每笔交易不同 |

**展示区间**（用户端）：

- 最低价 = `currentPrice`
- 最高价 = `round(currentPrice × (1 + dealPremiumPct))`

用户兑换或购买前需保证可用贡献金 ≥ 最高价；实际扣款为服务端计算的 `dealPrice`。

### 2.4 手续费

- 来源：`config.fund.rules.marketFeeRate`（默认 5%），挂单时快照至 `nft_listing.fee_rate`。
- 二级市场：`fee = round(dealPrice × feeRate)`，`sellerIncome = dealPrice - fee`。
- 一级兑换：平台收入，无卖家分成。

---

## 三、一级兑换

1. 用户进入藏品商城/详情，看到当前参考价、成交区间、近 30 日走势。
2. `POST /api/nft/:id/exchange`：校验库存/限购/余额 → 计算 `dealPrice` → 扣 `available_fund` → 写 `nft_exchange` 流水 → 发放 `user_nft`。

---

## 四、二级 C2C 交易

### 4.1 挂单

- `POST /api/nft/listings`：仅需 `userNftId`，**无需卖家输入售价**。
- 系统以挂单时刻 `current_price` 写入 `nft_listing.price`（参考价快照）。
- 需实名（可配置）、藏品状态为 `holding`。

### 4.2 购买

- `POST /api/nft/trade/:listingId/buy`：按 **2.3 公式** 计算 `dealPrice`（非挂单快照价）。
- 买家扣 `available_fund`（`nft_trade_buy`）；转移 `user_nft` 归属；卖家 `withdrawable_cash` 增加，**不写 fund_record**。
- `nft_trade` 记录：`price`（实际成交价）、`reference_price`、`deal_premium_factor`。

### 4.3 撤单 / 同步价

- `DELETE /api/nft/listings/:id`：撤单，藏品恢复 `holding`。
- `PUT /api/nft/listings/:id`：同步当前参考价至挂单快照（原「改价」）。

## 五、后台配置（`config` 表 `group=nft`, `key=market`）

| 配置项 | 字段 | 默认值 | 说明 |
| --- | --- | --- | --- |
| 开放交易 | `enabled` | `true` | 二级市场开关 |
| 日波动幅度 | `dailyFluctuationPct` | `0.05` | ±5% |
| 成交溢价比例 | `dealPremiumPct` | `0.1` | 最高溢价 10% |
| 挂单价区间 | `minPrice` / `maxPrice` | 1 / 99999 | 参考价校验 |
| 需实名 | `requireKyc` | `true` | 挂单/购买前置 |
| 手续费 | `fund.rules.marketFeeRate` | `0.05` | 贡献金规则中配置，NFT 页只读 |

管理入口：`/admin/nft`（藏品发行 + 市场配置）。

---

## 六、接口一览

| 接口 | 说明 |
| --- | --- |
| `GET /api/nft` | 列表：含 `startPrice`、`currentPrice`、`dealPriceMin/Max` |
| `GET /api/nft/:id` | 详情 + `priceHistory`（近 30 日） |
| `GET /api/nft/:id/price-history?days=30` | 价格走势 |
| `POST /api/nft/:id/exchange` | 兑换，返回 `dealPrice` |
| `POST /api/nft/listings` | 挂单（无 price 参数） |
| `PUT /api/nft/listings/:id` | 同步参考价 |
| `POST /api/nft/trade/:listingId/buy` | 购买，返回实际 `dealPrice` |
| `GET/PUT /api/admin/nft/market` | 波动/溢价等配置 |
| `POST /api/admin/nft` | 发行，`startPrice` 必填 |

---

## 七、与贡献金的关系

- 兑换/购买：消耗**可用贡献金**，详见 [贡献金业务说明.md](./贡献金业务说明.md)。
- 卖家成交所得：入**提现金**，不出现在贡献金明细。
- 成交记录：查 `nft_trade`；价格历史：查 `nft_price_history`。
